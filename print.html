Vibecoding voor Michiel Sectie 1 van 7

1 Wat is vibecoding?

De manier waarop je websites maakt is voorgoed veranderd.

Michiel, stel je voor: jij beschrijft in gewoon Nederlands wat je wilt maken, en kunstmatige intelligentie schrijft alle code voor je. Dat is vibecoding. Geen ingewikkelde programmeertalen, geen cursussen, geen frustratie over puntkomma's. Jij vertelt, Claude bouwt.

Jij bent de regisseur, Claude is de programmeur

Denk aan een film. De regisseur hoeft niet zelf de camera te bedienen, het licht te zetten of te acteren. Maar de regisseur bepaalt wat er gebeurt, hoe het eruitziet en wanneer het klaar is. Dat ben jij. Claude is je complete filmcrew — razendsnel, nooit moe, en altijd bereid om het opnieuw te proberen.

Waarom is dit een gamechanger?

Vroeger had je maanden nodig om te leren programmeren voordat je iets zinnigs kon bouwen. Nu heb je twee dingen nodig:

  • Een idee — wat wil je maken?
  • De vaardigheid om dat helder te beschrijven — en dat kun jij al
Hoe het werkt
Je typt in gewoon Nederlands wat je wilt. Claude leest dat, begrijpt het, en schrijft de code. Je kunt het resultaat meteen bekijken in je browser, feedback geven, en Claude past het aan. Net zo lang tot je tevreden bent.

Superpowers: de slimme aanpak

Vibecoding wordt pas echt krachtig als je gestructureerd werkt. Dat noemen we “superpowers” — een bewezen werkwijze om met Claude samen te werken:

  1. Brainstormen — je vertelt Claude je idee, Claude stelt slimme vragen, samen maak je het concreet
  2. Plannen — Claude maakt een bouwplan: wat wordt het, hoe ziet het eruit, wat zijn de stappen
  3. Bouwen — Claude schrijft de code stap voor stap, jij bekijkt het resultaat

Net als een architect die eerst schetst, dan technische tekeningen maakt, en dan pas gaat bouwen. Je springt niet van idee naar kant-en-klaar product — je doorloopt een slim proces.

Tip voor Michiel
Begin niet meteen met je droomproject. Start met iets kleins: een simpele webpagina. Zodra je doorhebt hoe het werkt (en dat gaat snel!), kun je steeds ambitieuzere dingen bouwen.
Vibecoding voor Michiel Sectie 2 van 7

2 Wat heb je nodig?

Vier stappen en je bent klaar om te beginnen.

  1. Claude Code downloaden en installeren Ga naar claude.ai en maak een account aan (als je dat nog niet hebt). Download vervolgens Claude Code — dat is de versie die op je computer draait en direct bestanden kan aanmaken.

    Je hebt ook tokens nodig — dat is het tegoed waarmee Claude werkt. Start klein (het goedkoopste abonnement), je kunt altijd upgraden als je meer gaat bouwen.
  2. GitHub account aanmaken Ga naar github.com en maak een gratis account aan. GitHub is de plek waar je site straks online komt te staan. Je hoeft er verder niets mee te doen — Claude regelt het.
  3. Werkmap aanmaken Maak een map aan waar je project in komt:
    # Windows:
    C:\mijn-project

    # Mac:
    ~/mijn-project
    Dit is de thuisbasis van je project. Alles wat Claude maakt komt hier terecht.
  4. Terminal openen en Claude starten
    # Windows: open Command Prompt of PowerShell
    > cd C:\mijn-project
    > claude

    # Mac: open Terminal
    $ cd ~/mijn-project
    $ claude
    Dat is het. Claude start op en wacht op je eerste opdracht.
Belangrijk
Zorg dat je in de juiste map staat als je Claude start. Claude werkt in de map waar je hem opstart — daar maakt hij bestanden aan en daar zoekt hij naar bestaande bestanden.
Vibecoding voor Michiel Sectie 3 van 7

3 Je eerste gesprek met Claude

Gewoon Nederlands praten. Echt, zo simpel is het.

Je eerste oefening

Claude draait. Je cursor knippert. Typ dit:

> Maak een HTML-pagina met de tekst "Hallo Wereld" in grote blauwe letters, gecentreerd op de pagina.

Claude maakt een bestand aan (bijv. index.html). Open dat bestand in je browser en je ziet het resultaat. Gefeliciteerd — je eerste webpagina!

Gewoon Nederlands werkt

Je hoeft geen technische termen te kennen. Zeg gewoon wat je wilt:

  • “Maak de achtergrond donkerblauw”
  • “Voeg een foto toe bovenaan de pagina”
  • “Maak een knop die naar google.nl gaat”

Claude begrijpt het en past de code aan. Niet goed? Zeg wat er anders moet. Net als tegen een collega.

Referentiemateriaal gebruiken

Zet bestanden in je werkmap die Claude mag gebruiken: afbeeldingen, teksten, Excel-bestanden, PDF’s. Zeg dan:

> Gebruik de bestanden in mijn map als basis voor de website.

Permissions: wat Claude wel en niet mag

Claude vraagt soms toestemming voordat hij iets doet (bijv. bestanden aanmaken). Dit zijn je opties:

Allow
Eenmalig goedkeuren. Claude vraagt de volgende keer opnieuw.
Allow always
Altijd goedkeuren voor dit type actie. Scheelt klikken.
Deny
Weigeren. Claude doet het niet en zoekt een alternatief.
Dangerous mode
Alles zonder vragen. Alleen als je Claude helemaal vertrouwt.
Tip
Begin met Allow zodat je ziet wat Claude doet. Als je vertrouwen hebt, schakel dan over naar Allow always voor de veelvoorkomende acties.

Resultaat bekijken

Open het HTML-bestand in je browser (dubbelklik op het bestand, of sleep het naar je browser). Niet tevreden? Ga terug naar Claude en zeg wat er anders moet. Dit heen-en-weer is normaal — het is precies hoe vibecoding werkt.

4 Superpowers — Brainstormen

De slimste manier om van vaag idee naar concreet plan te komen.

Wat zijn superpowers?

Superpowers is een gestructureerde manier om met Claude te werken. In plaats van meteen “bouw me een website” te zeggen, doorloop je een slim proces. Het resultaat: betere websites, minder gedoe, en je krijgt precies wat je in gedachten had.

De eerste stap is altijd brainstormen. Hier maak je samen met Claude je idee concreet.

Zo start je een brainstorm

> /superpowers:brainstorming

Ik wil een website maken voor mijn tennisvereniging.
Met een welkomstpagina, een kalender voor toernooien,
en een pagina waar leden zich kunnen aanmelden.

Wat er dan gebeurt

Claude gaat niet meteen bouwen. In plaats daarvan stelt Claude slimme vragen:

  • “Hoeveel leden heeft de vereniging ongeveer?”
  • “Moet de kalender bewerkbaar zijn, of alleen leesbaar?”
  • “Heb je al een logo of huisstijlkleuren?”
  • “Moeten leden kunnen inloggen?”

Jij beantwoordt de vragen. Soms met detail, soms met “maakt me niet uit, kies jij maar.” Allebei prima.

Het resultaat: jouw blauwdruk

Na het brainstormen maakt Claude een samenvatting: wat wordt het, voor wie, welke pagina's, welke functies, hoe ziet het eruit. Dit is je blauwdruk — het goedgekeurde ontwerp waar Claude mee aan de slag gaat.

Waarom dit werkt
Door eerst te brainstormen voorkom je dat Claude iets bouwt wat je niet bedoelde. Het is het verschil tussen “bouw me een huis” en “bouw me een huis met 3 slaapkamers, een open keuken, en een tuin op het zuiden.” Hoe beter de briefing, hoe beter het resultaat.
Tip voor Michiel
Je hoeft niet alle antwoorden te weten. “Weet ik niet, wat raad jij aan?” is een prima antwoord. Claude denkt mee en stelt alternatieven voor.

5 Van plan naar werkende site (en meer)

Claude bouwt, jij bekijkt en stuurt bij. Zo simpel is het.

Claude maakt automatisch een bouwplan

Na de brainstorm heeft Claude een blauwdruk. Nu maakt Claude daar automatisch een bouwplan van: een lijst stappen die hij gaat uitvoeren. Je ziet die stappen voorbijkomen en kunt op elk moment ingrijpen.

Stap voor stap bouwen

Claude bouwt je site onderdeel voor onderdeel. Na elke stap kun je het resultaat bekijken in je browser. Je ziet de site groeien:

  1. Eerst de basisstructuur
  2. Dan de opmaak en kleuren
  3. Vervolgens de inhoud
  4. Tot slot de details en afwerking

Aanpassen is normaal

Iets niet naar je zin? Gewoon zeggen:

> De titel mag groter en in het rood.
> Kun je de foto links uitlijnen in plaats van gecentreerd?
> Voeg nog een pagina toe met contactgegevens.

Dit heen-en-weer is geen bug — het is een feature. Zo werk je samen met Claude tot het precies goed is.

Meer dan websites!

Vibecoding is niet alleen voor websites. Claude kan veel meer:

Excel analyseren
“Analyseer de Excel in mijn map en maak er een samenvatting van met grafieken.”
PowerPoint maken
“Maak er een PowerPoint van met 10 slides voor mijn presentatie volgende week.”
Rapporten samenvatten
“Lees deze drie PDF's en maak er een samenvatting van op 1 A4.”
Data omzetten
“Zet deze CSV om naar een mooie overzichtstabel als webpagina.”

Zolang je het kunt beschrijven, kan Claude het waarschijnlijk maken. Probeer het gewoon!

6 Live op GitHub Pages

Je site online zetten is letterlijk een zin.

Gratis online met GitHub Pages

GitHub Pages is een gratis dienst van GitHub waarmee je een website online kunt zetten. Geen hosting kopen, geen technisch gedoe. Je site krijgt een adres zoals:

michielkamma.github.io/mijn-site

Zo doe je dat

Als je site klaar is en er goed uitziet in je browser, zeg je tegen Claude:

> Zet dit live op GitHub Pages.

Dat is het. Claude regelt de rest:

  • Maakt een repository aan op GitHub (als die er nog niet is)
  • Upload al je bestanden
  • Configureert GitHub Pages
  • Geeft je de link naar je live site

Eenmalig inloggen

De eerste keer vraagt Claude je om in te loggen bij GitHub. Dit gaat via de terminal — je krijgt een code die je op github.com invoert. Daarna onthoudt je computer de inloggegevens.

Let op
Je GitHub-gebruikersnaam wordt onderdeel van je websiteadres. Kies dus een nette naam bij het aanmaken van je account!

Later aanpassen

Wil je later iets wijzigen? Simpel:

  1. Open je terminal
  2. Ga naar dezelfde map (cd C:\mijn-project)
  3. Start Claude (claude)
  4. Zeg wat je wilt aanpassen
  5. Zeg “Zet de update live”

Claude herkent dat er al een site is en werkt die bij. Je hoeft niet opnieuw te beginnen.

Pro tip
Je kunt ook een eigen domeinnaam koppelen aan GitHub Pages (bijv. www.mijnsite.nl). Vraag het gewoon aan Claude als je dat wilt.

7 Cheatsheet

Knip uit, leg naast je toetsenbord, en ga aan de slag.

Eenmalige setup

  • Claude Code installeren + tokens kopen op claude.ai
  • GitHub account aanmaken op github.com (gratis)

Werkwijze per project

Map maken → claude → /superpowers:brainstorming → Bouwen → GitHub Pages

Handige zinnen

  • Gebruik de bestanden in mijn map — Claude leest je referentiemateriaal
  • Maak er een PowerPoint van — genereer een presentatie
  • Analyseer deze Excel — data-analyse en samenvatting
  • Zet dit live op GitHub Pages — site online zetten
  • Maak een samenvatting op 1 A4 — documenten condenseren
  • Verander de kleuren naar blauw en wit — stijl aanpassen

Problemen?

  • Claude doet niets: sta je in de juiste map? Check met cd C:\mijn-project
  • Resultaat klopt niet: gewoon zeggen wat er fout is, Claude past aan
  • Foutmelding: kopieer de fout en plak hem in Claude, hij lost het op
  • Opnieuw beginnen: wis de map, maak een nieuwe, start Claude

Terminal openen

  • Windows: zoek “Command Prompt” of “PowerShell” in Start
  • Mac: zoek “Terminal” via Spotlight (Cmd + Spatie)
